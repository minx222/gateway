# nestjs cas网关
## 技术站
+ nestjs
+ fastify
+ mongodb
+ redis

## 项目背景
### BFF
```text
随着微服务架构的普及，企业应用变得越来越分散，每个微服务通常专注于单一功能，这导致了前端应用程序需要与多个后端服务通信才能完成一个业务流程。这种模式下，直接从前端调用多个后端服务不仅增加了网络延迟，还可能引入安全风险和性能瓶颈。使用BFF能优先聚合后端服务，减少网络延迟，提高性能。
```

## 支持的单点登录模式
### CAS
```text
Cas是一种单点登录协议，本项目主要是基于Cas + 前端中台 + 后端微服务架构，实现单点登录。
```
